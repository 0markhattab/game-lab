Drawing on Canvas

Now that you have a loop, you need something to draw. Canvas gives you a 2D rendering context — basically a robot painter that knows shapes, colors, and transforms. Master ten methods and you can render anything.

02.1

The coordinate system

Canvas coordinates start at (0, 0) in the top-left. X goes right, Y goes down (yes, this trips up everyone who took geometry class). A 480×320 canvas has bottom-right corner at (480, 320).

Y is flipped from math class. If you want a rectangle 50 pixels from the top, that's y = 50. If you want it 50 from the bottom, that's y = H - 50.
02.2

The big six methods

These are the canvas drawing methods you'll use in 99% of every game you ever write:

// 1. Filled rectangle
ctx.fillRect(x, y, width, height);

// 2. Outlined rectangle
ctx.strokeRect(x, y, width, height);

// 3. Set the current fill / stroke color
ctx.fillStyle = '#9eff5a';
ctx.strokeStyle = '#ff3eb5';

// 4. Circle (uses the path API)
ctx.beginPath();
ctx.arc(x, y, radius, 0, Math.PI * 2);
ctx.fill();

// 5. Line
ctx.beginPath();
ctx.moveTo(x1, y1);
ctx.lineTo(x2, y2);
ctx.stroke();

// 6. Text
ctx.font = '24px monospace';
ctx.fillText('Hello', x, y);

Order matters. Canvas draws in the order you call methods — later draws cover earlier ones. The sky has to come before the moon, or the moon disappears. This is called the painter's algorithm.
02.4

Transforms (rotate, scale)

To rotate or scale something, you don't transform the shape — you transform the canvas before drawing, then restore it.

ctx.save();              // remember current state
ctx.translate(100, 100); // move origin to (100, 100)
ctx.rotate(Math.PI / 4); // rotate 45°
ctx.fillRect(-20, -20, 40, 40); // draw centered on new origin
ctx.restore();           // undo the translate + rotate

save / restore always come in pairs. Forget the restore and the next thing you draw is also rotated, then the next, then the next — until your whole scene spirals.
