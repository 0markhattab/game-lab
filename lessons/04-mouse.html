Module 04 · Input

Mouse & Cursor

The mouse opens up a whole genre of games — point-and-click adventures, target shooters, RTS games, drag-and-drop puzzles. The trick is converting screen coordinates to canvas coordinates correctly.

04.1

The events

Mouse events fire on the canvas itself (not on window):

canvas.addEventListener('mousemove', e => { /* track position */ });
canvas.addEventListener('mousedown', e => { /* click pressed */ });
canvas.addEventListener('mouseup',   e => { /* click released */ });
canvas.addEventListener('click',     e => { /* full click */ });
04.2

The coordinate trap

e.clientX and e.clientY give you the mouse position relative to the browser window, not your canvas. And if your canvas is being displayed at a different size than its internal resolution (very common with responsive layouts), the numbers won't even line up. The fix:

canvas.addEventListener('mousemove', e => {
  const r = canvas.getBoundingClientRect();
  mouse.x = (e.clientX - r.left) * (canvas.width  / r.width);
  mouse.y = (e.clientY - r.top)  * (canvas.height / r.height);
});

This is already wired up in every editor on this site — you can just read mouse.x and mouse.y.
