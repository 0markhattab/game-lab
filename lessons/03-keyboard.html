Keyboard Input

A game where the player can't do anything is just an animation. In this lesson you'll capture key presses, build the famous "keys object" pattern, and move a character around with WASD.

03.1

The two events

The browser fires two keyboard events you care about:

window.addEventListener('keydown', e => {
  console.log('pressed:', e.key);
});

window.addEventListener('keyup', e => {
  console.log('released:', e.key);
});

e.key gives you a friendly name like "a", "ArrowLeft", " " (spacebar), or "Escape".

The naive trap: beginners try to do player.x += 5 directly inside the keydown handler. This breaks because keydown fires once on press, then waits a moment, then auto-repeats — giving you laggy, stuttery movement. Don't do this.
03.2

The keys-object pattern

The right way: keep an object that tracks which keys are currently held down, then read from it each frame.

const keys = {};

window.addEventListener('keydown', e => keys[e.key] = true);
window.addEventListener('keyup',   e => keys[e.key] = false);

// inside update()
if (keys['ArrowLeft'])  player.x -= speed;
if (keys['ArrowRight']) player.x += speed;

Now movement is silky smooth — the loop checks the key state 60 times per second and moves accordingly. The keys object is already set up for you in every editor on this site.

03.4

Common key names

Some common e.key values worth memorizing:

Letters:        'a', 'b', 'c', ... (lowercase)
Arrow keys:     'ArrowUp', 'ArrowDown', 'ArrowLeft', 'ArrowRight'
Spacebar:       ' ' (a literal space)
Enter:          'Enter'
Escape:         'Escape'
Shift:          'Shift'
Numbers:        '0', '1', ..., '9'
