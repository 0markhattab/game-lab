Module 06 · Physics

Collision Detection

Two objects are touching. Now what? Detection — knowing when shapes overlap — is the foundation of every game mechanic that involves "hitting" something. We'll cover the two collision shapes you'll use in 95% of all games.

06.1

AABB: rectangle vs rectangle

AABB stands for "Axis-Aligned Bounding Box." Fancy name for: two non-rotated rectangles. The check is four comparisons:

function aabb(a, b) {
  return a.x < b.x + b.w &&
         a.x + a.w > b.x &&
         a.y < b.y + b.h &&
         a.y + a.h > b.y;
}

Read it out loud: "A's left edge is past B's right edge? AND A's right edge is past B's left? AND..." If all four are true, the rectangles overlap. If even one is false, they don't. This is the single most-used collision function in game dev.

The classic bug: using <= instead of <. With <=, two rectangles touching exactly at the edge count as colliding. Usually you don't want that. Stick to strict < and >.

The "test before you commit" pattern. Don't move the player and then check. Calculate the new position, test it for collisions, and only commit if it's clear. And solve X and Y separately — that's what lets you slide along a wall instead of getting stuck on it.
06.3

Circle vs circle

For circular objects (bullets, balls, characters with rounded hitboxes), use distance:

function circleHit(a, b) {
  const dx = a.x - b.x;
  const dy = a.y - b.y;
  const distSq = dx*dx + dy*dy;
  const rSum = a.r + b.r;
  return distSq < rSum * rSum;
}

We compare squared distances to avoid the expensive Math.sqrt call. Same answer, faster math. When you're checking 100 bullets against 100 enemies every frame (= 10,000 checks), this matters.
