The Game Loop

Every game on Earth — Mario, Fortnite, Minecraft, Pong — runs on the same fundamental pattern: a loop that updates the world and redraws the screen, 60 times per second. Once you understand it, every game stops being magic.

01.2

The pattern

A game loop is shockingly simple. It looks like this in pseudo-code:

function loop() {
  update();   // move things, check input, run game logic
  draw();     // clear the screen and redraw everything
  requestAnimationFrame(loop); // schedule next frame
}
loop();
Why the split? If you draw inside your update logic, your game becomes impossible to debug, pause, or run at different speeds. Keep them separate from day one — every studio in the world does.

Why requestAnimationFrame and not setInterval?

Two reasons. First, requestAnimationFrame syncs to the monitor's refresh rate (usually 60Hz), so animations look smooth. Second, when the user switches tabs, it pauses automatically — saving battery and preventing your game from running invisibly in the background. setInterval does neither.

01.4

Going deeper: delta time

The simple loop above runs at whatever speed the browser fires. On a 60Hz monitor, that's 60 frames per second. On a 144Hz gaming monitor, the same code runs 2.4× faster. That's a real bug. The fix is delta time:

let lastTime = 0;
function loop(currentTime) {
  const dt = (currentTime - lastTime) / 1000; // seconds since last frame
  lastTime = currentTime;

  update(dt);  // pass dt into update
  draw();
  requestAnimationFrame(loop);
}

Now instead of x += 2 ("move 2 pixels per frame"), you write x += 120 * dt ("move 120 pixels per second"). Same speed on every machine. We'll use this pattern in later lessons — for now, the simple version is fine.
